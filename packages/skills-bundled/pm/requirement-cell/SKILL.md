# 需求五格（PM 真源）

## 何时使用

- 写需求、评审需求、与研发对齐改动范围
- `chatMode=prd` 时收成右侧清单；explore 模式也可用于结构化说明

## 五格（每条需求）

| 格 | 内容 |
|----|------|
| 在哪个页面 | 业务页面名（非文件路径为主） |
| 改哪里 | 按钮/字段/列表/弹窗等具体触点 |
| 现在怎样 | 读代码后用业务话描述现状 |
| 希望改成 | 可验收的期望行为（分点） |
| 怎么验收 | 可操作的测试步骤 |

## 原则

- **主**：上述五格由 PM 定稿；验收、扯皮只认主
- **次**：研发附录（前后端线索）导出时 lazy 生成，非每轮必写
- 缺口标出，不连问；对话能短就短
- 现状必须 grep/read 核实，禁止编造

## 与工具

- PRD 模式：用 `update_requirement_draft` 写入右侧清单
- 需要代码线索：`get_anchor_preview`、grep/read，但不替代五格业务语言
