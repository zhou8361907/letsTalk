# AI Requirement OS — 使用指南

本文说明如何在本机安装、配置、启动 **ai-core**，以及如何通过工作台与 API 分析 Vue + Spring Boot 业务页面。

开发进度、阶段计划与取舍记录见：**[开发日志](./开发日志.md)**  
V1 的 JSON / Markdown / Prompt 规范见：**[V1页面分析规范](./V1页面分析规范.md)**  
V1 的接口与示例见：**[V1页面分析接口与样例](./V1页面分析接口与样例.md)**

## 1. 项目是什么

**ai-core** 是「AI 需求操作系统」的控制平面，面向企业常见的 **CRUD 列表页**：

- 扫描前端 Vue 页面与后端 Spring Boot 结构
- 为**单个页面**组装最小上下文（关联 `.vue`、API、Controller 等）
- 生成页面需求文档与结构化页面分析结果
- 可选：调用 DeepSeek 等 LLM 补全/润色文档
- 预留 Agent 会话、任务队列等扩展能力

默认自带样例工程路径（相对仓库根目录）：

`letsTalk/test-project/RunningAccount-master`

### 1.1 当前产品路线

当前建议将项目拆成三阶段推进，而不是一开始把重心放在沙箱和自动改码闭环上：

1. **V1：页面级扫描与说明书**
   目标是以“单个页面”为单位，稳定输出前后端联动信息，让使用者第一时间看懂页面职责、结构、接口、规则与关联文件。
2. **V2：页面问答 Agent**
   在 V1 的页面上下文之上，增加记忆、工具调用、后台扫描代码、跨文件追踪与多轮问答能力。
3. **V3：代码修改闭环**
   最后再增强为可生成代码计划、预览 diff、应用修改、回滚修改，并在需要时接入真实前端沙箱预览。

这意味着：

- **近期主线**：把“看懂页面”做到可靠
- **Agent**：先会查、会解释、会追踪，再考虑直接改代码
- **沙箱**：暂时降级为辅助能力，不作为当前核心

## 2. 环境要求

| 项目 | 要求 |
|------|------|
| Python | 3.10 及以上 |
| 包管理 | [uv](https://docs.astral.sh/uv/)（推荐） |
| 被分析项目 | Vue 前端目录 + Spring Boot 后端目录（路径可配置） |
| LLM（可选） | DeepSeek API Key；无 Key 时仍可做扫描与规则解析，AI 补全会退回草稿模式 |

## 3. 安装

在仓库中进入 `ai-core` 目录：

```bash
cd /path/to/letsTalk/ai-core
uv sync
```

开发依赖（测试、ruff、mypy）会一并安装。仅运行服务时执行 `uv sync` 即可。

## 4. 配置

### 4.1 环境变量（`.env`）

复制示例并填写密钥：

```bash
cp .env.example .env
```

编辑 `.env`：

```env
DEEPSEEK_API_KEY=你的密钥
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_BASE_URL=https://api.deepseek.com

# 可选：Agent 默认路径（也可在工作台里填）
AIRO_FRONTEND_ROOT=
AIRO_BACKEND_ROOT=
AIRO_DEFAULT_PAGE=
```

| 变量 | 说明 |
|------|------|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥；**不配置时**「AI 补全文档」不会调用模型，返回本地草稿 |
| `DEEPSEEK_MODEL` | 模型名，默认 `deepseek-chat` |
| `DEEPSEEK_BASE_URL` | OpenAI 兼容接口地址 |
| `AIRO_FRONTEND_ROOT` | 默认前端根目录（绝对路径） |
| `AIRO_BACKEND_ROOT` | 默认后端根目录（绝对路径） |
| `AIRO_DEFAULT_PAGE` | 默认分析的 Vue 文件路径 |

> **注意**：请勿将含真实密钥的 `.env` 提交到 Git。

### 4.2 Agent 配置（`config/agent.toml`，可选）

高级行为（多角色模型、记忆、工具开关等）通过 TOML 配置：

```bash
cp config/agent.example.toml config/agent.toml
```

按需修改 `workspace_root`、`paths`、`models`、`tools` 等。若不存在 `config/agent.toml`，程序使用内置默认值，并用 `.env` 覆盖模型与路径。

配置约定：

- **`agent.toml`**：结构与非敏感默认值
- **`.env`**：密钥、模型名、默认路径

## 5. 启动服务

### 5.1 开发模式（推荐）

热重载，改代码自动重启：

```bash
./scripts/run_workbench.sh
```

默认监听：**http://127.0.0.1:8000**

如果本机端口很多，建议在 `.env` 里统一配置：

```bash
AIRO_API_HOST=127.0.0.1
AIRO_API_PORT=8000
AIRO_FRONTEND_PORT=8081
AIRO_SANDBOX_BASE_URL=http://127.0.0.1:8081
```

### 5.2 生产式启动

```bash
AIRO_API_HOST=0.0.0.0 AIRO_API_PORT=8000 ./scripts/run_workbench.sh
```

### 5.3 健康检查

```bash
curl http://127.0.0.1:8000/health
# {"status":"ok"}
```

## 6. 使用工作台（Web UI）

浏览器打开：

**http://127.0.0.1:8000/workbench**

根路径 `/` 会自动跳转到工作台。

### 6.1 界面布局

| 区域 | 作用 |
|------|------|
| 左侧 · 项目配置 | 项目名称、前端路径、后端路径 |
| 左侧 · 页面列表 | 扫描得到的 Vue 页面，点击选中 |
| 左侧 · 关联文件 | 当前页面相关源码文件 |
| 中间 · 结构化 JSON | 页面分析结果的机器可读主产物 |
| 中间 · Markdown 报告 | 基于 JSON 渲染的人类可读报告 |
| 中间 · 实际对话框 | 预留给 V2 的多轮问答入口，当前仅占位 |
| 中间 · 沙箱 | 默认折叠，当前仅保留入口，不作为主线 |

### 6.2 推荐操作流程

1. **载入样例**（左上角「载入」）  
   自动填充默认样例项目的前端/后端绝对路径。

2. **扫描页面**（「扫描页面」）  
   调用项目发现：列出 Vue 视图、API 模块、Controller、DTO 等。

3. **选择页面**  
   在左侧列表点击某一个 `.vue` 文件。

4. **加载页面工作区**（「加载页面工作区」）  
   基于源码做规则解析，展示关联文件与页面基础草稿。**此步不消耗 LLM。**

5. **AI 生成页面分析结果**（当前主目标）  
   后续将由 Agent 读取页面上下文，生成结构化 JSON，再由 JSON 渲染 Markdown 报告。  
   当前阶段这条链路还在收敛中，目标是先把单页结果稳定下来。

6. **沙箱与代码修改能力**  
   当前不作为主流程，默认只保留入口和后续扩展位置。

### 6.4 当前阶段建议怎么用

如果当前以 **V1 页面说明书** 为主，建议优先使用下面这条链路：

1. 载入样例 / 填写自己的项目路径
2. 扫描页面
3. 选择一个页面
4. 加载页面工作区
5. 阅读页面背景、结构、表格、弹窗、接口、关联文件
6. 如有需要，再看结构化 JSON 和 Markdown 报告

这时可以先忽略沙箱、代码应用、回滚等能力，把工作台当作“页面级说明书生成器”来使用。

页面打开时会自动执行：载入样例 → 扫描。若样例路径不存在，请在侧栏手动填写**绝对路径**后再扫描。

### 6.3 分析自己的项目

在侧栏填写：

- **项目名称**：任意标识
- **前端路径**：Vue 工程根目录（含 `src/views` 等），**建议绝对路径**
- **后端路径**：Spring Boot 工程根目录（含 `controller` 等），**建议绝对路径**

然后「扫描页面」→ 选页 → 「加载页面工作区」。

路径需在本机真实存在；扫描与解析均读取本地文件，不会上传代码到第三方（除你主动点击「AI 补全文档」时，关联文件片段会进入 LLM 请求）。

## 7. HTTP API 速查

Base URL：`http://127.0.0.1:8000`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| GET | `/workbench` | 工作台页面 |
| GET | `/api/sample-project` | 默认样例项目配置 |
| POST | `/api/analyze/discovery` | 项目结构发现 |
| POST | `/api/page-workspace?page_path=...` | 单页工作区（规则解析 + Schema） |
| POST | `/api/llm/page-documentation?page_path=...` | LLM 增强页面文档 |
| GET | `/example/page-ir` | 示例 PageIR JSON |
| GET | `/example/runtime-schema` | 示例运行时 Schema |
| GET | `/api/agent/manifest` | Agent 能力清单 |
| POST | `/api/agent/session` | 创建 Agent 会话 |
| POST | `/api/agent/tasks` | 排队 Agent 任务 |

### 7.1 请求体示例（发现 / 工作区）

`POST /api/analyze/discovery`  
`Content-Type: application/json`

```json
{
  "project_name": "my-erp",
  "frontend_path": "/abs/path/to/vue",
  "backend_path": "/abs/path/to/springboot",
  "entry_pages": []
}
```

单页工作区在同一 body 上增加查询参数：

```bash
curl -X POST "http://127.0.0.1:8000/api/page-workspace?page_path=/abs/path/to/views/foo/index.vue" \
  -H "Content-Type: application/json" \
  -d '{"project_name":"my-erp","frontend_path":"/abs/path/to/vue","backend_path":"/abs/path/to/springboot","entry_pages":[]}'
```

### 7.2 OpenAPI 文档

服务启动后访问：

- Swagger UI：**http://127.0.0.1:8000/docs**
- ReDoc：**http://127.0.0.1:8000/redoc**

## 8. 开发与测试

```bash
# 运行测试
uv run pytest

# 代码检查
uv run ruff check src tests
uv run mypy
```

若本地已有 `.env` 且配置了 `DEEPSEEK_API_KEY`，个别测试（如「缺 Key 应报错」）可能受环境影响；CI 中应使用隔离环境变量。

如果工作台能打开，但中间的沙箱 iframe 打不开，优先检查：

1. `AIRO_SANDBOX_BASE_URL` 是否指向真实前端 dev server
2. 前端是否以 `VUE_APP_MOCK=true npm run dev -- --port 8081` 之类的方式启动
3. 该端口是否被别的项目占用

## 9. 常见问题

### Q：扫描结果为空？

- 确认 `frontend_path` 指向 Vue 工程且存在 `src/views`（或项目内视图目录）
- 使用**绝对路径**，避免相对路径因工作目录不同而失效

### Q：「AI 补全文档」无效果或显示 draft？

- 检查 `.env` 中 `DEEPSEEK_API_KEY` 是否配置正确
- 查看右侧「AI 输出状态」中的 `warnings`

### Q：加载工作区报 400？

- `page_path` 必须是发现结果中的某个 Vue 文件路径
- 路径需可读、编码为 UTF-8（非 UTF-8 文件会忽略错误继续）

### Q：样例项目路径不对？

默认样例在 `letsTalk/test-project/RunningAccount-master`。若未克隆该目录，请在工作台手动填写你自己的前后端路径。

### Q：如何换模型或 Provider？

当前 LLM 入口统一走 DeepSeek（OpenAI 兼容）。修改 `config/agent.toml` 中 `[models.primary]` 及 `.env` 中的 `DEEPSEEK_*`；扩展其他 Provider 需改 `llm/gateway.py`（尚未内置）。

## 11. 相关里程碑（了解边界）

当前版本已较好支持：**发现 → 单页工作区 → 规则文档 → 沙箱预览 →（可选）LLM 文档**。

仍在规划或占位的能力：

- 自然语言需求 → 页面 Schema Patch
- 结构化需求 Diff（`diff/` 包）
- 完整 Agent 多轮对话与持久化记忆
- 沙箱内真实组件渲染（当前为结构草图）

更多架构说明见项目根目录 `README.md`。
