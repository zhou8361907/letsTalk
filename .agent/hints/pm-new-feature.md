# 新功能评估指南

当你在代码中找不到已有实现（asIs 为空或仅有入口），**且** `PM_PRD_RULES` 中的触发器已激活时，按本指南评估。

> **核心原则**：你是资深 PM，不是速记员。新功能场景下你的价值是帮 PM 想清楚"做什么、怎么做、影响谁"，而不是追问 PM 技术细节。

---

## 一、合理性与 MVP 提案（静默思考，向 PM 提建议，禁止反问）

### 1. 查重自检（静默 grep）
- grep 功能关键词匹配现有页面/菜单/按钮，确认不是重复建设
- 若已有功能覆盖 → 直接告知 PM「现有功能 X 已覆盖」，无需评估

### 2. 低成本替代方案（静默思考 → 主动提案）
- 思考能否用**配置化**（加字典项/参数配置）解决？
- 思考能否用**加字段**（在现有表/页面加字段）解决？
- 思考能否用**改参数**（调整现有配置/阈值）解决？
- **能则在回复中主动向 PM 提出低成本方案，不反问**

### 3. 界定 MVP（主动建议，禁止反问）
- 基于你对系统的了解，判断实现复杂度
- **不要问 PM"MVP 边界在哪"**，你要主动根据实现复杂度标注建议：
  - `[MVP建议] 建议一期先做固定报表+核心指标（覆盖 80%），实时推送+拖拽配置放二期`
  - `[MVP建议] 建议一期先做单笔操作，批量能力看使用反馈再迭代`
- MVP 结论写入 openQuestions，标注 `[MVP建议]`

## 二、复用性扫描（禁止全局乱搜，采用降维搜索）

按以下优先级 grep，找到最近似的东西作为起点，**不要从零设计**：

### 1. 找入口页面
```
grep -r '功能关键词' workFront/src/pages/ --include='*.vue' -l
```
重点找：路由配置、菜单配置、el-table/el-form 布局

### 2. 找后端接口
```
grep -r '模块名\|资源名' workBack/*/src/main/java --include='*.java' -l
```
重点找：`@RequestMapping`、`@RestController`、`Controller`

### 3. 找跨模块相似能力
不同业务线（smsbm/smbe/smip/smrm）经常有高度相似的查询/审批/导出逻辑。
```
grep -r '逻辑关键词' workBack/smsbm workBack/smbe workBack/smip --include='*.java' -l
```

### 4. 找基础设施
- 报表引擎：现有 ECharts/table 组件模式
- 审批流：现有工作流配置（zfjh 类型）
- 银行通道：现有 smsbm 支付接口
- 附件/导入导出：现有工具类

### 5. 复用结论
- 找到可复用的 → codePaths 写"参考 X 模块的 Y 接口/页面"，而非生造
- 没找到可复用的 → 正常，标「全新模块，无直接复用参考」

## 三、影响范围核查（6 维度 checklist）

| 维度 | 核查问题 | 输出格式 |
|------|----------|----------|
| **数据流** | 新增字段/表影响现有报表、统计图表、导出 Excel？需同步修改关联模块查询？ | `[影响] 需同步修改 smrm 月度统计报表` |
| **权限安全** | 需新增权限点？现有菜单角色需重新赋权？数据按什么范围隔离？接口鉴权/防越权？ | `[影响] 需新增菜单权限点：fund_monitor_view` |
| **跨模块配置** | smip 需新增省份差异化配置？银行适配器需新增渠道或修改参数？ | `[影响] smip 需配四川/宁夏差异化参数` |
| **性能容量** | 批量操作上限？分页设计？超时处理？数据量级预估？Redis/DB 支撑？ | `[影响] 首次加载建议限最近 12 个月` |
| **兼容回滚** | 旧数据向后兼容？上线出问题回滚方案？需灰度开关控制？ | `[影响] 需加 feature.toggle 控制，默认关闭` |
| **安全审计** | 登录鉴权？防越权？敏感字段脱敏？关键操作记审计日志？ | `[影响] 跨省转账操作需记 audit_log` |

**核查结论**：影响范围结论写入 openQuestions，标注 `[影响]`

---

## 四、readyToFinalize 豁免

新功能场景下：
- **asIs 空** → 正常现象，不加分惩罚
- **codePaths 空** → 允许，但须标注 `[复用参考路径]`
- **必须输出** `[MVP建议]` + `[影响范围结论]` 才能标记 finalize
- 缺 acceptance 不影响 finalize（PM 需后续补充）
