# 业务提示（供产品经理 / Agent 参考）

本目录由**人维护**，记录「业务说法 → 页面或代码路径」的映射。

## 治理规则

- **谁可以写：** 你（开发者）和 PM 用户可以直接编辑这些文件
- **Agent 只读，不能自动写：** Agent 发现 hints 过时时，会在对话中提出修改建议，由你确认后手动更新
- **权威性：** 内容可能过时，**与代码冲突时以 workFront / workBack 实际代码为准**

## 使用方式

- Agent 在「写需求」模式下会看到本目录文件列表；需要细节时用 `read` 打开对应 md。
- PM 描述需求涉及某个模块时，Agent 应主动用 `get_business_hints` 浏览列表，判断是否需要 `read` 具体 hint。

## 现有文件

| 文件 | 主题 | 适用场景 |
|------|------|----------|
| `module-overview.md` | SMIFC 模块全景与依赖关系 | 了解各模块职责、前后端映射、跨模块调用 |
| `banking-domain.md` | 银行对接领域知识与支付流水线 | 涉及银行交互、支付、对账的需求 |
| `regional-differences.md` | 省份差异与多区域部署机制 | 涉及多省份覆盖、区域配置的需求 |
| `common-pitfalls.md` | 开发常见陷阱与注意事项 | 任何需求，作为检查清单参考 |
| `data-model.md` | 核心数据模型与数据流 | 涉及字段、表结构、数据流转的需求 |

## 文件命名

- 一个主题一个文件
- 建议包含：业务名词、关联页面路径、关键规则、常见误区
