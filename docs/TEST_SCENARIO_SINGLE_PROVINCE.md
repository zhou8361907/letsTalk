# 测试场景 3：宁夏个性化 — 支付计划手工支付加审批层级

## 场景背景

宁夏之前做过一些 smip 定制。现在宁夏医保局要求：手工支付（N001 批次那种）原来科室主任审完就能付，现在超过 50 万的必须再加一级分管领导审批。

其他省份不动，只有宁夏有这个要求。

## 对话流程

### 第 1 轮 — 抛出需求

> "宁夏那边说他们手工支付现在审批层级不够，超过 50 万的单子只有科室主任审了就付了，风险太大。要求加一级分管领导审批。只改宁夏，其他省不动。"

**期望 Agent 行为：**
- ✅ 调 `get_business_hints` → 看到 `banking-domain.md`、`regional-differences.md`
- ✅ `read regional-differences.md` → 了解宁夏已有 smip 定制
- ✅ 识别「手工支付」→ 关联 N001 批次、`P4.startsWith("N001")` 逻辑
- ✅ 识别「超过 50 万」→ 追问：是单笔超 50 万还是累计？触发后整个批次等领导审还是只拦超额的？
- ✅ 追问：领导审批是系统内审批还是线下签完再录？
- ❌ 不猜其他省份也改

### 第 2 轮 — PM 补充

> "单笔超过 50 万的拦住，没超的继续走。拦住的整批等领导审完再走。领导审批走系统内，现在审批流程就有两级审批的配置，银川那边在用。宁夏之前做过定制，应该可以参考银川的那个审批配置。"

**期望 Agent 行为：**
- ✅ 读代码验证已有两级审批配置
- ✅ 识别「可参考银川配置」→ 读 smip 下银川的审批配置
- ✅ 区分配置项变动 vs 代码变动
- ✅ 追问：宁夏这个新审批层级是加在 N001 特有流程上，还是通用审批链上？
- ✅ 识别「超过 50 万拦住，未超继续」→ 拆分逻辑变更（审批分流）
- ✅ asIs 写清楚宁夏目前手工支付只有一级审批

## 预期最终产物

**1 条需求条目**（个性化需求，不应拆成多条）：

- page: 手工支付审批（宁夏定制）
- control: 审批层级配置
- asIs: 宁夏手工支付（N001 批次）只有科室主任一级审批，超 50 万也无额外控制
- toBe: 1）单笔 ≥ 50 万的 N001 批次触发二级审批；2）触发后整批等待分管领导审批；3）参照银川现有两级审批配置实现
- codePaths: smsbm（手工支付）、smip/ningxia/（宁夏定制）、smip/yinchuan/（银川审批配置参考）

## 测试重点

| 考察点 | 判断方式 |
|--------|----------|
| **hints 触发** | 是否调 `get_business_hints` → `read regional-differences.md` |
| **单省定制理解** | 是否明确「只改宁夏，其他省不动」 |
| **代码验证** | 是否读银川审批配置确认「可参考」 |
| **阈值追问** | 是否追问「50 万是单笔还是累计」「超额的批次怎么处理」 |
| **审批 vs 拆分** | 是否理解这是审批链扩展，不是拆分逻辑变更 |
